Installare Qiskit¶

Requisiti¶

Qiskit supporta Python 3.6 o versioni successive. Tuttavia, dato che sia Python che Qiskit sono ecosistemi in continua evoluzione, a volte, quando si verificano nuovi rilasci in uno o nell’altro, potrebbero sorgere problemi di compatibilità.

Se si riscontrano difficoltà durante l’installazione o l’utilizzo di Qiskit a seguito di un aggiornamento di Python, controllare i metadati del pacchetto Qiskit alla voce Programming Language per verificare quale specifiche versioni di Python sono supportate.

Suggeriamo di installare Anaconda, una distribuzione multi-piattaforma di Python per il calcolo scientifico. Jupyter, incluso in Anaconda, è consigliato per interagire con Qiskit.

Qiskit è stato testato ed è supportato dai seguenti sistemi a 64-bit:

  • Ubuntu 16.04 o successivi

  • macOS 10.12.6 o successivi

  • Windows 7 o successivi

Installazione¶

Si consiglia di utilizzare gli ambienti virtuali di Python, per separare Qiskit da altre applicazioni e migliorare la propria esperienza di utilizzo.

Il modo migliore per utilizzare gli ambienti virtuali è tramite il comando conda, incluso in Anaconda. Un ambiente Conda permette di selezionare una specifica versione di Python ed un insieme specifico di librerie. Apri una finestra del terminale nella cartella dove desideri lavorare.

È consigliato utilizzare il prompt Anaconda che viene installato automaticamente insieme ad Anaconda stessa. È sufficiente creare un ambiente virtuale in Anaconda ed attivarlo. Indipendentemente dal sistema operativo utilizzato, che sia Windows o una macchina Linux, i seguenti comandi possono essere eseguiti direttamente dal prompt Anaconda.

Crea un ambiente minimale dove hai installato unicamente Python.

conda create -n name_of_my_env python=3

Attiva il nuovo ambiente.

conda activate name_of_my_env

Nota

Se stai utilizzando versioni di conda precedenti alla 4.6, utilizza: Su Linux o macOS: source activate name_of_my_env Su Windows: activate name_of_my_env

Installa poi il pacchetto Qiskit, che include Terra, Aer, Ignis e Aqua.

pip install qiskit

Nota

A partire da Qiskit 0.13.0 è necessario pip 19, o una versione più recente, per poter installare qiskit-aer su Linux a partire da file binari pre-compilati. Se non hai pip 19 installato puoi eseguire pip install -U pip per aggiornarlo. Senza pip 19 o più recente, questo comando cercherà di installare qiskit-aer da sdist (distribuzione sorgente), che proverà in sottofondo a compilare aer localmente.

Se i pacchetti sono stati installati correttamente, per vedere la lista di pacchetti attivi nell’ambiente virtuale si può utilizzare il comando conda list.

Se si intende utilizzare la funzionalità di visualizzazione o i notebook Jupyter si consiglia di installare qiskit con i requisiti extra di visualizzazione

pip install qiskit[visualization]

Vale la pena precisare che se si utilizza zsh (la shell di default nelle nuove versioni di macOS) bisognerà inserire qiskit[visualization] tra virgolette:

pip install 'qiskit[visualization]'

Nota

Dopo aver installato e verificato i pacchetti di Qiskit che si desidera usare, importali nel tuo ambiente Python per iniziare a lavorare.

import qiskit

Nota

Se vuoi contribuire alla comunità di Qiskit sviluppando e sottomettendo codice partendo dalla versione più recente del codice di Qiskit, guarda Costruire pacchetti Qiskit dal sorgente.

Accesso ai sistemi di IBM Quantum¶

IBM Quantum mette a disposizione di tutti, attraverso IBM Quantum Lab, sia diversi computer quantistici reali, sia simulatori di calcolo classici ad alta performance. Segui questi passaggi per configurare il tuo ambiente Qiskit e poter inviare job ai sistemi di IBM Quantum.

Per configurare il tuo account, devi creare un file locale di configurazione che includa una chiave API

1. Crea un account gratuito IBM Quantum.

2. Copia il tuo token di accesso dalla dashboard di IBM Quantum.

3. Esegui i seguenti comandi per salvare il tuo token API localmente e poterlo successivamente utilizzare nel file di configurazione chiamato qiskitrc. Sostituisci MY_API_TOKEN con il token API che hai salvato nel tuo editor di testo.

from qiskit import IBMQ
IBMQ.save_account('MY_API_TOKEN')

Per maggiori dettagli, ad esempio su come gestire le credenziali di diversi account di IBM Quantum, fare riferimento a questo tutorial chiamato The IBM Quantum Account.

Verifica della Versione Installata¶

Dato che il pacchetto Qiskit include un insieme di diversi elementi, visualizzare la versione eseguendo il comando qiskit.__version__ può causare confusione, poichè il risultato riporta unicamente la versione del pacchetto qiskit-terra. Questo accade in quanto il namespace qiskit in Python non deriva dal pacchetto Qiskit, ma fa invece parte del pacchetto qiskit-terra.

import qiskit
qiskit.__version__

'0.16.4'

Per ottenere le versioni di tutti gli elementi di Qiskit installati nel tuo ambiente puoi usare l’attributo __qiskit_version__. Ad esempio, eseguendo il seguente comando si ottiene un dizionario che include le versioni per ciascuno dei pacchetti Qiskit installati.

qiskit.__qiskit_version__

{'qiskit-terra': '0.16.4',
 'qiskit-aer': '0.7.6',
 'qiskit-ignis': '0.5.2',
 'qiskit-ibmq-provider': '0.12.2',
 'qiskit-aqua': '0.8.2',
 'qiskit': '0.24.1'}

Suggerimento

Se stai riportando una issue o hai bisogno di condividere la versione di Qiskit che hai installato, puoi utilizzare l’attributo __qiskit_version__.
