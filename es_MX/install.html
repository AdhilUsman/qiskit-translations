Instalación de Qiskit¶

Requisitos¶

Qiskit es compatible con versiones de Python 3.6 o superior. Sin embargo, tanto Python como Qiskit son sistemas en constante evolución, y debido a esto, cuando se lanzan nuevas actualizaciones en alguno de los dos entornos puede que ocurran problemas de compatibilidad.

Si tiene problemas al instalar o utilizar Qiskit después de actualizar Python, compruebe los metadatos del paquete Qiskit en la sección de Programming Language para ver qué versiones específicas de Python están soportadas.

Recomendamos instalar Anaconda, una distribución de Python multiplataforma para la computación científica. Jupyter, incluido en Anaconda, es recomendado para interactuar con Qiskit.

Qiskit está probado y soportado en los siguientes sistemas de 64 bits:

  • Ubuntu 16.04 o posterior

  • macOS 10.12.6 o posterior

  • Windows 7 o posterior

Instalación¶

Recomendamos utilizar entornos virtuales de Python para separar Qiskit de otras aplicaciones y mejorar su experiencia.

La forma más sencilla de usar los entornos es usando el comando conda, incluido con Anaconda. Un entorno Conda permite especificar una versión específica de Python y un conjunto de bibliotecas. Abra una ventana de terminal en el directorio donde desea trabajar.

Es preferible que utilice el prompt de Anaconda instalado con Anaconda. Todo lo que tienes que hacer es crear un entorno virtual dentro de Anaconda y activarlo. Estos comandos se pueden ejecutar en el prompt de Anaconda independientemente de la máquina (Windows o Linux).

Cree un entorno mínimo que sólo tenga Python instalado en él.

conda create -n name_of_my_env python=3

Active el entorno.

conda activate name_of_my_env

Nota

Si está utilizando versiones conda anteriores a la 4.6, use: Para Linux o macOS: source activate name_of_my_env, para Windows: activate name_of_my_env

A continuación, instale el paquete Qiskit, que incluye Terra, Aer, Ignis y Aqua.

pip install qiskit

Nota

A partir de Qiskit 0.13.0 pip 19 o posterior se necesita para instalar qiskit-aer desde el binario precompilado en Linux. Si no tiene instalado pip 19, puede ejecutar pip install -U pip para actualizarlo. Sin pip 19 o posterior, este comando intentará instalar qiskit-aer desde sdist (distribución fuente) que intentará compilar un aer localmente de forma oculta.

Si los paquetes se instalaron correctamente puedes utilizar el comando «conda list» para ver los paquetes activos en tu entorno virtual.

Si tiene la intención de utilizar la funcionalidad de visualización o los notebooks de Jupyter, se recomienda instalar qiskit con los requisitos adicionales de visualización

pip install qiskit[visualization]

Vale la pena mencionar que si utilizas zsh (que es el shell predeterminado en versiones más recientes de macOS), necesitaras poner «qiskit[visualization]», entre comillas:

pip install 'qiskit[visualization]'

Nota

Después de haber instalado y verificado los paquetes de Qiskit que quiera usar, impórtelos a su entorno con Python para comenzar a trabajar.

import qiskit

Nota

Si desea contribuir a la comunidad Qiskit desarrollando y aportando código con el código Qiskit actualizado más reciente, ver Construir paquetes Qiskit desde la fuente.

Acceso a los Sistemas Cuánticos de IBM¶

IBM Quantum ofrece varias computadoras cuánticas reales y simuladores clásicos de alto rendimiento a través de IBM Quantum Lab. Siga los siguientes pasos para configurar su entorno Qiskit para enviar trabajos a los sistemas de IBM Quantum.

Para configurar tu cuenta, crearás un archivo de configuración local que incluye una clave API

1. Cree una cuenta gratis de IBM Quantum.

2. Copia tu token de acceso desde el tablero de IBM Quantum.

3. Ejecute los siguientes comandos para almacenar su token de API localmente para su uso posterior en un archivo de configuración llamado qiskitrc. Sustituya MY_API_TOKEN por el valor del token de API que guardó en su editor de texto.

from qiskit import IBMQ
IBMQ.save_account('MY_API_TOKEN')

Para obtener más detalles, como por ejemplo de cómo gestionar las credenciales de varias cuentas de IBM Quantum, consulte este tutorial llamado ` La cuenta de IBM Quantum <https://github.com/Qiskit/qiskit-tutorials/blob/master/qiskit/fundamentals/3_the_ibmq_account.ipynb>`__.

Verificando la versión instalada¶

Teniendo en cuenta que el paquete de Qiskit incluye una constelación de diferentes elementos, sólo con imprimir la versión ejecutando qiskit.__version__ puede llevar a confusión porque sólo devuelve la versión del paquete de qiskit-terra. Esto es porque el espacio de nombres de qiskit en Python no viene del paquete de Qiskit sino que es parte del paquete de qiskit-terra.

import qiskit
qiskit.__version__

'0.16.4'

Para ver las versiones de todos los elementos de Qiskit en su entorno puede utilizar el atributo __qiskit_version__. Por ejemplo, al ejecutar el siguiente comando se obtendrá un diccionario que incluye las versiones para cada uno de los paquetes Qiskit instalados.

qiskit.__qiskit_version__

{'qiskit-terra': '0.16.4',
 'qiskit-aer': '0.7.6',
 'qiskit-ignis': '0.5.2',
 'qiskit-ibmq-provider': '0.12.2',
 'qiskit-aqua': '0.8.2',
 'qiskit': '0.24.1'}

Truco

Si está registrando un problema o necesita compartir sus versiones instaladas de Qiskit para algo, use el atributo __qiskit_version__.
