மேம்பட்ட VQE பயன்பாடு¶

மேலும் முன்னேற்றப்பட்ட ` VQE <https://qiskit.org/documentation/stubs/qiskit.aqua.algorithms.VQE.html>` __ திறன்களை பயன்படுத்துவதற்கு பல அளபுருக்கள் இருக்கிறது. இந்த டுடோரியல் `` initial_point``, expectation மற்றும் `` gradient`` போன்ற அளவுருக்களை உள்ளடக்கும்.

மேட்ரிக்ஸ் தயாரிப்பு நிலை முறையுடன் (Matrix Product State method) ஏர் பயன்படுத்துவது போன்ற மேம்பட்ட சிமுலேட்டர் பயன்பாட்டையும் இது உள்ளடக்கும்.

[1]:

from qiskit import BasicAer
from qiskit.aqua.operators import X, Z, I
from qiskit.aqua import QuantumInstance, aqua_globals
from qiskit.aqua.algorithms import VQE
from qiskit.aqua.components.optimizers import SLSQP
from qiskit.circuit.library import TwoLocal

இங்கே நாம் ஒரே இயக்குநர் பயன்படுத்தப்படும் பிற VQE படிவத்தில் பயனுள்ளதாகும்.

[2]:

H2_op = (-1.052373245772859 * I ^ I) + \
        (0.39793742484318045 * I ^ Z) + \
        (-0.39793742484318045 * Z ^ I) + \
        (-0.01128010425623538 * Z ^ Z) + \
        (0.18093119978423156 * X ^ X)

ஆரம்ப புள்ளி¶

` `initial_point ` ` அளபுருக்கள், கொடுக்கப்பட்ட புள்ளியை துவக்க அனுமதிக்கிறது, எங்கே புள்ளி மாறுபாட்டின் பட்டியல் உள்ளது. முன்னிருப்பாக ஆரம்ப புள்ளி ` ` எதுவும் ` ` இல்லையென்றால், VQE ஒன்றை தேர்ந்தெடுக்க இயலாது. இந்த வழக்கில் தேர்வு செய்யப்பட்ட வேறுபாடு வடிவம் ஒரு விருப்பமான புள்ளியைக் கொண்டிருப்பதால், ஆரம்ப நிலையின் குற அடிப்படையில், அது தேர்ந்தெடுக்கப்படும், இல்லையென்றால், எந்த விகிதங்களுடன் ஒழுங்குபடுத்தும் ஒரு நிலையான ஆரம்ப புள்ளி தேர்ந்தெடுக்கப்படும். ஒரு ஆரம்ப புள்ளி கொடுக்கப்பட்டால் அது முன்னுரிமையை எடுத்துக் கொள்ளும்.

ஆரம்ப புள்ளியை ஏன் பயன்படுத்த வேண்டும்? சிக்கலுக்கான ஒரு நியாயமான தொடக்க புள்ளியை நீங்கள் யூகித்திருந்தால் அல்லது முந்தைய பரிசோதனையிலிருந்து தகவல் இருந்தால் ஒருவேளை ஒரு காரணம்.

பயன்பாட்டை நிரூபிக்க, முதலில் ஒரு 'algorithms introduction <01_algorithms_introduction.ipynb # A-complete-working-example> `__ டுடோரியலில் இருந்து ஒரு தீர்வின் உகந்த புள்ளியைப் பெறுவதற்கான முதல் வேலை உதாரணத்தை மீண்டும் செய்வோம்.

[3]:

from qiskit.aqua import aqua_globals
seed = 50
aqua_globals.random_seed = seed
qi = QuantumInstance(BasicAer.get_backend('statevector_simulator'), seed_transpiler=seed, seed_simulator=seed)

ansatz = TwoLocal(rotation_blocks='ry', entanglement_blocks='cz')
slsqp = SLSQP(maxiter=1000)
vqe = VQE(operator=H2_op, var_form=ansatz, optimizer=slsqp, quantum_instance=qi)
result = vqe.run()

import pprint
pp = pprint.PrettyPrinter(indent=4)
pp.pprint(result)

ஆரம்ப புள்ளி வழங்கப்படாதபோது (default of None) ஒரு சீரற்ற மதிப்பிலிருந்து தொடங்கும் போது `` optimizer_evals`` 10 க்கு எதிராக 72 ஆக இருக்கும் அந்த முடிவு மிக விரைவாக வந்ததை இங்கே காண்கிறோம்.

எடுத்துக்காட்டுகளுக்கு இது பயனுள்ளதாக இருக்கும் இடத்தில், ஒரு பிரச்சினைக்கான தீர்வை மிக நெருக்கமான ஒத்த பிரச்சினைக்கான தீர்வுக்கான யூகத்திற்கு பயன்படுத்தலாம். வேதியியல் மிகச் சிறந்த எடுத்துக்காட்டு, அங்கு ஒரு விலகல் சுயவிவரத்தைத் திட்டமிட மூலக்கூறின் இடை-அணு தூரத்தை (களை) மாற்றுகிறோம். தூர மாற்றங்கள் சிறியதாக இருக்கும்போது, தீர்வு இன்னும் முந்தையதை அருகில் இருக்கும் என்று எதிர்பார்க்கிறோம். ஒரு நுட்பம் ஒரு தீர்விலிருந்து உகந்த புள்ளியை அடுத்த கட்டத்திற்கான தொடக்க புள்ளியாகப் பயன்படுத்துவது. இப்போது மிகவும் சிக்கலான நுட்பங்கள் சாத்தியமானவை, அவை முந்தைய தீர்வை நேரடியாகப் பயன்படுத்துவதற்குப் பதிலாக முந்தைய தீர்வை (களை) அடிப்படையாகக் கொண்டு ஒரு ஆரம்ப நிலையை கணக்கிட சில விரிவாக்கங்களைச் செய்கின்றன. Qiskit Chemistry sampling_potential_energy_surfaces <../chemistry/05_Sampling_potential_energy_surfaces.ipynb#Extrapolation> __ டுடோரியல் அத்தகைய பூட்ஸ்ட்ராப்பிங் மற்றும் எக்ஸ்ட்ராபோலேஷனைக் காட்டுகிறது.

எதிர்பார்ப்பு¶

VQE பணிபுரியும் ஹாமில்டோனிய ஆபரேட்டரின் ஆற்றல் அளவுருவாக்கப்பட்ட மாறுபட்ட வடிவத்துடன் மதிப்பிடும்போது எதிர்பார்ப்பு மதிப்பு. எதிர்பார்ப்பு மதிப்பைக் கணக்கிட VQE ஒரு expectation <https://qiskit.org/documentation/apidoc/qiskit.aqua.operators.expectations.html> __ பொருளின் ஒரு உதாரணத்தைப் பயன்படுத்துகிறது. அத்தகைய நிகழ்வு expectation அளவுரு வழியாக வழங்கப்படலாம் அல்லது இயல்புநிலை வழக்கில், அது None என்ற மதிப்பைக் கொண்டிருக்கலாம், VQE `ExpectationFactory <https://qiskit.org/documentation/stubs/qiskit.aqua.operators.expectations.ExpectationFactory.html> `__ வழங்கப்பட்ட பின்தளத்தில் அடிப்படையில் ஒரு பொருத்தமான நிகழ்வை உருவாக்க.

பெரும்பாலான சந்தர்ப்பங்களில் VQE ஒரு பொருத்தமான நிகழ்வை உருவாக்க அனுமதித்தால் போதுமானது. இருப்பினும், Qiskit Aer qasm_simulator ஒரு ஸ்னாப்ஷாட் அறிவுறுத்தலை ஆதரிக்கிறது, இது ஆபரேட்டர் எதிர்பார்ப்பு கணக்கீட்டுடன் இணைந்து பயன்படுத்தப்படலாம். பயன்படுத்தினால், விளைவு சிறந்தது, அதாவது ஸ்டேட்வெக்டர் சிமுலேட்டரைப் போன்றது, மற்றும் ஷாட் சத்தம் இல்லை. மக்கள் பொதுவாக qasm_simulator ஐ குறைவான சத்தம் (மாதிரி சத்தம்) தேர்வு செய்வதால், மேலும் உண்மையான சாதன விளைவுகளைப் போலவே இருக்க வேண்டும் என்பதால், VQE இல் `` include_custom`` கொடி உள்ளது, இது எதிர்பார்ப்பு செயல்திறன் க்கு அனுப்பப்படுகிறது. ஏர் காஸ்ம் சிமுலேட்டரைப் பயன்படுத்தும் போது, இது True என அமைக்கப்பட்டால், தொழிற்சாலை ஸ்னாப்ஷாட் அறிவுறுத்தலைப் பயன்படுத்தும் AerPauliExpectation ஐத் தரும், False, இயல்புநிலை, பின்னர் வழக்கமான PauliExpectation.

பின்வரும் எடுத்துக்காட்டு `` include_custom = True`` ஐக் காட்டுகிறது, அங்கு விளைவு ஸ்டேட்வெக்டர் சிமுலேட்டருடன் பொருந்துகிறது. உண்மையில் ஸ்டேட்வெக்டர்_சிமுலேட்டரை நேரடியாகப் பயன்படுத்துவதை விட இதைச் செய்வது நல்லது / வேகமாக இருக்கும். ஏனென்றால், பிந்தைய சந்தர்ப்பத்தில், ஹாமில்டோனியன் பவுலிஸின் தொகையாக இருக்கும்போது அது மேட்ரிக்ஸ் வடிவமாக மாற்றப்பட வேண்டும், மேலும் இதில்_கஸ்டம் உண்மை எனும்போது செய்யப்படும் ஸ்னாப்ஷாட் அறிவுறுத்தலைப் பயன்படுத்தும்போது இது தவிர்க்கப்படுகிறது.

சாய்வு¶

ஒளிச்சேர்வு ஒரு கிரேடியன்-அடிப்படையிலான நுட்பத்தை பயன்படுத்தி ஒரு பயனர் வரையறுக்கப்பட்ட கிரேடியன்களுடன் கொடுக்கலாம். அவற்றின் முன்னிருப்பு சாய்வு கணிப்புகளுக்கு பதிலாக பயன்படுத்தப்படும். கிரேடியன் அதன் அளபுரு வழியாக ஒப்பீட்டாளர் மூலம் நேரடியாக கடத்தப்படுகின்றன.

`` gradient`` வழங்கப்பட்ட பயனரின் பயன்பாடு Monitoring VQE Convergence <02_vqe_convergence.ipynb#Using-Gradient-framework> __ டுடோரியலில் காண்பிக்கப்படுவதால், நான் உங்களை அங்கேயே குறிப்பிடுவேன். Gradients framework <../operators/02_gradients_framework.ipynb> __ டுடோரியலில் சாய்வுகளைப் பற்றி அதிகம் உள்ளது.

குவாண்டம் நிகழ்வு மற்றும் மேம்பட்ட உருவகப்படுத்துதல்¶

`` statevector_simulator`` அல்லது qasm_simulator அல்லது உண்மையான சாதன பின்தளத்தில் இருந்து உருவாக்கப்பட்ட குவாண்டம்இன்ஸ்டான்சனைக் கடந்து செல்வதில் நீங்கள் மிகவும் ஆர்வமாக இருக்கும்போது, பொருந்தும் போது ஏரின் மேம்பட்ட உருவகப்படுத்துதல் முறைகளையும் பயன்படுத்தலாம். உதாரணமாக, ஏர் Matrix Product State <../simulators/7_matrix_product_state_method.ipynb> __ முறையை நாம் எளிதாகப் பயன்படுத்தலாம், இது அதிக எண்ணிக்கையிலான குவிட்டுகளுக்கு அளவிடக்கூடிய ஆற்றலைக் கொண்டுள்ளது.
