போர்ட்ஃபோலியோ உகப்பாக்கம்¶

அறிமுகம்¶

தரவுகள் நிர்வாகத்தில், பரவலாக இரண்டு அணுகுமுறைகள் உள்ளன: செயலில் மற்றும் செயலற்ற முதலீட்டு மேலாண்மை. செயலற்ற முதலீட்டு நிர்வாகத்திற்குள், indexed-கண்காணிப்பு நிதிகள் உள்ளன மற்றும் போர்ட்ஃபோலியோ பல்வகைப்படுத்தலின் அடிப்படையில் அணுகுமுறைகள் உள்ளன, அவை குறைந்த எண்ணிக்கையிலான பிரதிநிதித்துவ பங்குகளால் அதிக எண்ணிக்கையிலான சொத்துக்களைக் கொண்ட ஒரு போர்ட்ஃபோலியோவை பிரதிநிதித்துவப்படுத்துவதை நோக்கமாகக் கொண்டுள்ளன. இந்த நோட்புக் ஒரு போர்ட்ஃபோலியோ classification சிக்கலை விளக்குகிறது, இது சமீபத்தில் இரண்டு காரணங்களுக்காக பிரபலமாகிவிட்டது: 1. இது ஒரு வரையறுக்கப்பட்ட பட்ஜெட்டுடன், வரையறுக்கப்பட்ட பரிவர்த்தனை செலவில் ஒரு குறியீட்டின் (அல்லது இதேபோன்ற பெரிய சொத்துக்களின்) செயல்திறனைப் பிரதிபலிக்க உதவுகிறது. அதாவது: பாரம்பரிய குறியீட்டு-கண்காணிப்பு குறியீட்டில் உள்ள அனைத்து சொத்துக்களையும் வாங்கக்கூடும், குறியீட்டில் உள்ள அதே எடையுடன். இது பல காரணங்களுக்காக நடைமுறைக்கு மாறானதாக இருக்கலாம்: ஒரு சொத்துக்கு ஒரு சுற்று நிறைய கூட நிர்வாகத்தின் கீழ் உள்ள சொத்துக்களை விட அதிகமாக இருக்கலாம், ஒருங்கிணைந்த கட்டுப்பாடுகளுடன் கூடிய குறியீட்டு-கண்காணிப்பு சிக்கலின் பெரிய அளவு தேர்வுமுறை சிக்கலை கடினமாக்குகிறது, மற்றும் குறியீட்டில் உள்ள எடைகளுக்கு நிலைகளை சரிசெய்ய அடிக்கடி மறுசீரமைப்பின் பரிவர்த்தனை செலவுகள் அணுகுமுறையை விலை உயர்ந்ததாக மாற்றக்கூடும். எனவே, ஒரு பிரபலமான அணுகுமுறை :math: q சொத்துகள் சந்தையை பிரதிநிதித்துவப்படுத்தும் :math:` n` சொத்துக்கள், எங்கே:math: q ஐ விட கணிசமாக சிறியது :math:` n`, ஆனால் போர்ட்ஃபோலியோ அடிப்படை சந்தையின் நடத்தையை பிரதிபலிக்கிறது. சொத்துக்களை எவ்வாறு குழுவாக்குவது என்பதை தீர்மானிக்க :math: q கொத்துகள் மற்றும் எது தீர்மானிக்க வேண்டும் :math:` q` சொத்துக்கள் பின்வருவனவற்றைக் குறிக்க வேண்டும் :math: q கொத்துகள் ஒரு பெரிய அளவிலான தேர்வுமுறை சிக்கலைத் தீர்ப்பதற்கு சமம். [கார்னூஜோல்ஸ் & டுட்டுங்கு, 2006] இல் அறிமுகப்படுத்தப்பட்ட போர்ட்ஃபோலியோ பல்வகைப்படுத்தல் சிக்கலுக்கான கணித மாதிரியை பின்வருவனவற்றில் விவரிக்கிறோம். பாரம்பரியமாக, நவீன போர்ட்ஃபோலியோ கோட்பாடு கோவாரன்ஸ் மேட்ரிக்ஸை சொத்துக்களுக்கு இடையிலான ஒற்றுமையின் ஒரு நடவடிக்கையாக கருதுகிறது என்பதைக் கவனியுங்கள். இருப்பினும், கோவாரன்ஸ் மேட்ரிக்ஸ் அபூரணமானது. உதாரணமாக, லண்டன் மற்றும் நியூயார்க்கில் பட்டியலிடப்பட்ட ஒரு நிறுவனத்தைக் கவனியுங்கள். இரண்டு பட்டியல்களும் மிகவும் ஒத்ததாக இருக்க வேண்டும் என்றாலும், இரண்டு பட்டியல்களின் விலைகளின் நேரத் தொடரின் பகுதிகள் மட்டுமே ஒன்றுடன் ஒன்று சேரும், ஏனெனில் சந்தைகள் திறக்கும் நேரங்களின் ஓரளவு ஒன்றுடன் ஒன்று. ஒத்திசைவுக்குப் பதிலாக, [பெர்ன்ட் மற்றும் கிளிஃபோர்ட், 1994] இன் டைனமிக் டைம் வார்பிங்கை இரண்டு நேரத் தொடர்களுக்கிடையிலான ஒற்றுமையின் ஒரு நடவடிக்கையாகக் கருதலாம், இது சில காலங்களுக்கு, தரவு ஒரே ஒருவரால் மட்டுமே கைப்பற்றப்படுகிறது என்பதை அனுமதிக்கிறது நேரத் தொடரின், மற்றவர்களுக்கு, இரு நேரத் தொடர்களும் பங்கு விலையின் இணையான பரிணாமத்தின் காரணமாக ஒற்றுமையை வெளிப்படுத்துகின்றன.

நாங்கள் நிரூபிக்கும் ஒட்டுமொத்த workflow அடங்கும்:

  1. நிலநடுத் தரவுகளை எடுக்கவும். நமது case ல் இது ஒரு சிறிய எண்ணிக்கை அமெரிக்க ஸ்டாக் ஆகும்.

  2. வலைப்பதிவுகளின் விகிதங்களின் பரிணாம வளர்ச்சி அளிக்கிறது. நமது வழக்கில், இது விக்கிபீடியா அல்லது நாச்டாக் அல்லது LSE அல்லது யூரோடெனில் இருந்து ஒழுங்குபடுத்தப்பட்ட தினசரி விலை தரவுகளின் ஒரு எளிதான ஏற்றுமதி உள்ளது, ஒவ்வொன்றும் ஒரு உண்மையான வர்த்தக நிர்வாகத்தில், ஒரு அதிக அதிக விகிதமாக கருதலாம்.

  3. நேர வரிசையில் ஜோடி-விதமான ஒத்திசைவு ஒழுங்குபடுத்துகிறது. நமது வழக்கில், நாம் ஒரு linear-time approximation of the dynamic time warping ஐ உபயோக்க்கிறோம், இன்னும் classical கணினியில்.

  4. :math:` q ` பிரதிநிதித்துப் சொத்துக்களின் உண்மையான துடுப்பாட்டப் போட்டிகளை ஒழுங்குபடுத்தி, ஒத்திசைவு அளவில் அடிப்படையாகக் கொண்டது. இந்த படி இரண்டு முறை இயங்குகிறது, உண்மையில். முதலில், நாம் ஒரு IBM solver (IBM ILOG CPLEX அல்லது Exact Eigensolver) ஒரு இயக்கி மதிப்பை பரிந்துரைக்கிறோம். இரண்டாவது, நாம் ஒரு மாற்று, hybrid கணினியில் சேர்த்து செயல்படுத்த வேன்டும்.

  5. முடிவுகளின் விவரம். நமது வழக்கில், இது மீண்டும் ஒரு எளிமையான plot உள்ளது.

பின்வருமாறு, நாம் முதலில் (4) மேலே பயன்படுத்தப்பட்ட model களை விளக்குகிறது, முன் தேவையான மற்றும் தரவு ஏற்றுக் கொள்ளும் நிறுவனத்துடன் தொடர்ந்து முன்னர்.

மாதிரி¶

[Cornuejols & Tutuncu, 2006] இல் பேசப்படுகையில், நாம் ஒரு கணித மாதிரியான ஒரு கணித மாதிரி என்று விவரித்துள்ளோம் ஒவ்வொரு குழுவிலிருந்து ஒவ்வொரு குழுவிலிருந்து ஒவ்வொரு குழுவிலிருந்து ஒரு பிரதிநிதி நிர்வாகத்தை தேர்ந்தெடுக்கிறது. மாதிரி பின்வரும் தகவல்களை அடிப்படையாகக் கொண்டு வருகின்றது, அதாவது நாம் பின் மேலும் விரிவாக்கத்தில் பரிந்துரைக்கப்படும்:

\[\rho_{ij} = \textrm{similarity}\, \textrm{between}\, \textrm{stock}\, i \, \textrm{and}\, \textrm{stock}\, j.\]

உதாரனத்திற்க்கு, \(\rho_{ii} = 1\), \(\rho_{ij} \leq 1\) for \(i \neq j\) and \(\rho_{ij}\) போன்ற பங்குகளுக்கு இவை பெரியது. இதற்கு ஒரு எடுத்துக்காட்டு, பங்குகளின் வருமானத்திற்கும் இடையேயான தொடர்பு \(i\) and \(j\). ஆனால் ஒருவர் மற்ற ஒற்றுமை குறியீடுகளை தேர்வு செய்யலாம் \(\rho_{ij}\).

நான் தீர்க்க ஆர்வமாக உள்ள சிக்கல்:

\[(M) \quad f = \max_{x_{ij}, y_{j}} \,\, \sum_{i=1}^n \sum_{j=1}^n \rho_{ij} x_{ij}\]

clustering கட்டுப்பாட்டில் பொருள்:

\[\sum_{j=1}^n y_j = q,\]

ஒழுங்கற்ற கட்டுப்பாடுகள்:

\[\sum_{j=1}^n x_{ij} = 1, \,\textrm{ for }\, i = 1,\ldots, n, \quad x_{ij} \leq y_j,\,\textrm{ for }\, i = 1,\ldots, n; \, j = 1,\ldots, n, \quad x_{jj} = y_j,\,\textrm{ for }\, j = 1,\ldots, n,\]

மற்றும் ஆங்கில கட்டுப்பாடுகள்:

\[\quad x_{ij}, y_j \in\{0,1\}, \,\textrm{ for }\, i = 1,\ldots, n; \, j = 1,\ldots, n.\]

Variables :math: y_j எந்த பங்குகள் (:math:` j` index நிதியில் உள்ளன என்பதை விவரிக்கிறது). ஒவ்வொரு பங்குக்கும்:math: i = 1 ,dots,n, variables :math:` x_ {ij} எந்தப் பங்கைக் குறிக்கிறது:math: index நிதியில் j` மிகவும் ஒத்துப் போகிறது:math: i `(:math: x_ {ij} = 1` if :math: j என்பது index நிதியில் மிகவும் ஒத்துப் போகும் பங்கு ,:math:` 0` இல்லையெனில்).

முதலாவது கட்டுப்பாட்டை :math:` q ` ஸ்டாக் நிர்வாகத்தில் தேர்ந்தெடுக்கிறது. இரண்டாவது கட்டுப்பாடுகள் :math:` i ` க்கு சரியாக ஒரு பிரதிநிதி ஸ்டாக் :math:` j . மூன்றாவது மற்றும் நான்காவது கட்டுப்பாடுகள் என்று உறுதிசெய்கிறது என்று ஸ்டாக் :math: i ` க்குள் :math:` j ` க்கு மட்டுமே :math:` j . மாதிரிகளின் உரிமைகள் :math: n ` ஸ்டாக்ஸ் மற்றும் அவர்களின் பிரதிநிதிகளுக்கும் இடையிலான ஒத்திசைவு பெரிதாக்குகிறது. வெவ்வேறு செலவு செயல்பாடுகள் கூட கருதப்பட முடியும்.

ஒரு திசையனில் முடிவு variables ஐ ஒன்றிணைப்போம்

\[{\bf z} = [x_{11},x_{12},\ldots,x_{11}, x_{22},\ldots,x_{nn}, y_{1},\ldots,y_{n}],\]

ஒருவரின் பரிமாணம் :math:` {bf z} in{0, 1} ^ N , with :math: N = n (n + 1) ` மற்றும் உகந்த தீர்வைக் குறிக்கவும் :math:` {bf z} ^ * , மற்றும் உகந்த செலவு :math: f ^ * `.

ஒரு hybrid அணுகுமுறை¶

ஃபார்ஹி, கோல்ட்ஸ்டோன் மற்றும் குட்மேன் (2014) ஆகியவற்றின் குவாண்டம் தோராயமான தேர்வுமுறை அணுகுமுறையைப் பின்பற்றி, கிளாசிக்கல் மற்றும் குவாண்டம் கம்ப்யூட்டிங் படிகளை இணைக்கும் அணுகுமுறையை இங்கே நாங்கள் நிரூபிக்கிறோம்.

Binary polynomial தேர்வுமுறையை உருவாக்குங்கள்¶

:math:` (M) ` இல் இருந்து :math: x_ {ij} q leq y_j சமத்துவமின்மை கட்டுப்பாடுகள் சமமான சமத்துவக் கட்டுப்பாடுகளுடன் மாற்றுவதன் மூலம் மட்டுமே சமத்துவக் கட்டுப்பாடுகளுடன் ஒரு binary constraints தேர்வுமுறையை உருவாக்க முடியும். :math:`x_{ij} (1-y_j) = 0. பிறகு இந்த பிரச்சனை இவ்வாறு போகும்:

\[(BPO) \quad f = \max_{x_{ij}, y_{j}} \,\, \sum_{i=1}^n \sum_{j=1}^n \rho_{ij} x_{ij}\]

clustering கட்டுப்பாடுகள், ஒருங்கிணைந்த கட்டுப்பாடுகள், மற்றும் பின்வரும் திருத்தப்பட்ட தொடக்க கட்டுப்பாடுகள்:

\[\sum_{j=1}^n x_{ij} = 1, \,\textrm{ for }\, i = 1,\ldots, n,\]
\[\quad x_{ij} (1- y_j) = 0,\,\textrm{ for }\, i = 1,\ldots, n; \, j = 1,\ldots, n,\]
\[\quad x_{jj} = y_j,\,\textrm{ for }\, j = 1,\ldots, n.\]

Ising Hamiltonian உருவாக்கு¶

நாம் இப்போது இசைங் ஹாமில்டோனியன் (QUBO) தண்டனி முறைகளால் உருவாக்க முடியும் (ஒவ்வொரு சமத்துவம் கட்டுப்பாட்டுக்களுக்கு ஒரு தண்டனையை அறிமுகம் செய்யவும் :math:` A `)

\[(IH) \quad H = \sum_{i=1}^n \sum_{j=1}^n \rho_{ij} x_{ij} + A\Big( \sum_{j=1}^n y_j - q\Big)^2 + \sum_{i=1}^n A\Big( \sum_{j=1}^n x_{ij} - 1\Big)^2 + \sum_{j=1}^n A (x_{jj}-y_j)^2 +\sum_{i=1}^n \sum_{j=1}^n A \left(x_{ij} (1- y_j)\right).\]

Hamiltonian இருந்து Quadratic Programming (QP) வடிவம்¶

வெக்டர் :math:` {bf z} `, Ising Hamiltonian elements பின்வருமாறு எழுதப்பட முடியும்,

முதல் term:

\[\sum_{i=1}^n \sum_{j=1}^n \rho_{ij} x_{ij} = [\rho_{11},\rho_{12},\ldots,\rho_{11}, \rho_{22},\ldots,\rho_{nn}|{\bf 0}_n ]{\bf z} =: {\bf c}_0^T {\bf z}\]

இரண்டாம் term:

\[\begin{split}A\Big( \sum_{j=1}^n y_j - q\Big)^2 = A \Big(\sum_{j=1}^n y_j\Big)^2 - 2 A \sum_{j=1}^n y_j + A q^2 = A {\bf z}^T \left[\begin{array}{c}{\bf 0}_{n^2} \\ \hline {\bf 1}_n \end{array}\right]\left[\begin{array}{cc}{\bf 0}_{n^2} | {\bf 1}_n \end{array}\right]{\bf z} - 2 A q [{\bf 0}_{n^2}|{\bf 1}_n]{\bf z} + A q^2 =: {\bf z}^T {\bf Q}_0 {\bf z} + {\bf c}_1^T {\bf z} + r_0\end{split}\]

மூன்றாம் term:

\[\sum_{i=1}^n A\Big( \sum_{j=1}^n x_{ij} - 1\Big)^2 = A\sum_{i=1}^n \Big(\sum_{j=1}^n x_{ij}\Big)^2 - 2 A \sum_{i=1}^n\sum_{j=1}^n x_{ij} + n A = \qquad\qquad\qquad\qquad\qquad\qquad\qquad\]

இது சமமானது:

\[\begin{split}\qquad\qquad\qquad\qquad\qquad\qquad\qquad = A {\bf z}^T \left(\sum_{i=1}^n \left[\begin{array}{c}{\bf 0}_{n(i-1)} \\ {\bf 1}_n \\ {\bf 0}_{n(n-i)} \\ \hline {\bf 0}_{n} \end{array}\right]\left[\begin{array}{cccc}{\bf 0}_{n(i-1)} & {\bf 1}_n & {\bf 0}_{n(n-i)} & | {\bf 0}_{n} \end{array}\right]\right){\bf z} - 2 A [{\bf 1}_{n^2}|{\bf 0}_n]{\bf z} + n A =: {\bf z}^T {\bf Q}_1 {\bf z} + {\bf c}_2^T {\bf z} + r_1\end{split}\]

நான்காவது term:

\[\begin{split}A \sum_{j=1}^n (x_{jj}-y_j)^2 = A {\bf z}^T \left(\sum_{j=0}^{n-1} \left[\begin{array}{c}{\bf 0}_{nj + j} \\ 1 \\ {\bf 0}_{n^2-(nj+j+1)} \\ \hline {\bf 0}_{j} \\ -1 \\ {\bf 0}_{n-j-1} \end{array}\right]\left[\begin{array}{cccccc}{\bf 0}_{nj + j} & 1 & {\bf 0}_{n^2-(nj+j+1)} & | {\bf 0}_{j} & -1 & {\bf 0}_{n-j-1} \end{array}\right]\right){\bf z} = A {\bf z}^T {\bf Q}_2 {\bf z}\end{split}\]

ஐந்தாம் term:

\[\begin{split}\sum_{i=1}^n \sum_{j=1}^n A \left(x_{ij} (1- y_j)\right) = A [{\bf 1}_{n^2}|{\bf 0}_n]{\bf z} + A {\bf z}^T \left( \sum_{i=1}^n \sum_{j=1}^n \left[\begin{array}{ccc|c} & & & \\ & {\bf 0}_{n^2\times n^2} & & -1/2_{(ij,j)} \\ & & & \\ \hline & -1/2_{(j, ij)} & & {\bf 0}_{n} \end{array}\right] \right) {\bf z} =: {\bf z}^T {\bf Q}_3 {\bf z} + {\bf c}_3^T {\bf z}\end{split}\]

எனவே, formulation இவ்வாறு,

\[(IH-QP)\quad \max_{{\bf z}\in\{0,1\}^{n(n+1)}} \, {\bf z}^T ({\bf Q}_0+{\bf Q}_1+ {\bf Q}_2 + {\bf Q}_3 ){\bf z} + ({\bf c}_0+{\bf c}_1+{\bf c}_2+{\bf c}_3)^T {\bf z} +r_0+r_1+r_2\]

இது வேற்றுமை கோட்பாடு variational quantum eigensolver க்கு pass செய்ய முடியும்.

class ClassicalOptimizer:
    def __init__(self, rho, n, q):

        self.rho = rho
        self.n = n  # number of inner variables
        self.q = q  # number of required selection

    def compute_allowed_combinations(self):
        f = math.factorial
        return int(f(self.n) / f(self.q) / f(self.n - self.q))

    def cplex_solution(self):

        # refactoring
        rho = self.rho
        n = self.n
        q = self.q

        my_obj = list(rho.reshape(1, n ** 2)[0]) + [0. for x in range(0, n)]
        my_ub = [1 for x in range(0, n ** 2 + n)]
        my_lb = [0 for x in range(0, n ** 2 + n)]
        my_ctype = "".join(['I' for x in range(0, n ** 2 + n)])

        my_rhs = [q] + [1 for x in range (0, n)] +[0 for x in range (0, n)] + [0.1 for x in range(0, n ** 2)]
        my_sense = "".join(['E' for x in range(0, 1+n)]) + "".join(['E' for x in range(0, n)]) + "".join(
            ['L' for x in range(0, n ** 2)])

        try:
            my_prob = cplex.Cplex()
            self.populatebyrow(my_prob, my_obj, my_ub, my_lb, my_ctype, my_sense, my_rhs)

            my_prob.solve()

        except CplexError as exc:
            print(exc)
            return

        x = my_prob.solution.get_values()
        x = np.array(x)
        cost = my_prob.solution.get_objective_value()

        return x, cost

    def populatebyrow(self, prob, my_obj, my_ub, my_lb, my_ctype, my_sense, my_rhs):

        n = self.n

        prob.objective.set_sense(prob.objective.sense.minimize)
        prob.variables.add(obj=my_obj, lb=my_lb, ub=my_ub, types=my_ctype)

        prob.set_log_stream(None)
        prob.set_error_stream(None)
        prob.set_warning_stream(None)
        prob.set_results_stream(None)

        rows = []
        col = [x for x in range(n**2, n**2+n)]
        coef = [1 for x in range(0, n)]
        rows.append([col, coef])

        for ii in range(0, n):
            col = [x for x in range(0+n*ii, n+n*ii)]
            coef = [1 for x in range(0, n)]

            rows.append([col, coef])

        for ii in range(0, n):
            col = [ii * n + ii, n ** 2 + ii]
            coef = [1, -1]
            rows.append([col, coef])

        for ii in range(0, n):
            for jj in range(0, n):
                col = [ii*n + jj, n ** 2 + jj]
                coef = [1, -1]

                rows.append([col, coef])

        prob.linear_constraints.add(lin_expr=rows, senses=my_sense, rhs=my_rhs)

Quantum Computing with IBM Q¶

குவாண்டம் தீர்வு, நாம் Qiskit பயன்படுத்துவோம். நாம் முதலில் ஒரு வகுப்பு QuantumOptimizer வரையறுக்கிறோம், அந்த பிரச்சனையை தீர்க்க குவாண்டம் அணுகல் குறியீடு மற்றும் பிறகு அதை நாம் அதை தீர்க்க மற்றும் அதை தீர்க்க. நாம் வகுப்புக்குள் பின்வரும் முறைகளை வரையறுக்கிறோம்:

  • exact_solution : அதை உறுதிப்படுத்த Ising Hamiltonian \(Z\) basis சரியாக குறியாக்கம் செய்யப்பட்டுள்ளதா என்றும், அதன் eigendecomposition ஐ நாம் கிளாசிக்கலாக கணக்கிடலாம், அதாவது, பரிமாணத்தின் சமச்சீர் அணியைக் கருத்தில் கொள்ளுங்கள் \(2^N \times 2^N\). கையில் உள்ள பிரச்சினைக்கு \(n=3\), அதாவது \(N = 12\), பல மடிக்கணினிகளுக்கான வரம்பாகத் தெரிகிறது;

  • ` vqe_ தீர்வு ` : தீர்வு :math: (M) ` வழியாக variational quantum eigensolver (VQE) `;

  • ` qaoa_தீர்வு ` : பிரச்சனை :math: (M) ` ஒரு Quantum Approximate Optimization Algorithm (QAOA) மூலம் தீர்மானிக்கிறது.

        return self.decode_result(result)

    def vqe_solution(self):
        qubitOp = portfolio_diversification.get_operator(self.rho, self.n, self.q)
        backend = Aer.get_backend('statevector_simulator')
        seed = 50
        cobyla = COBYLA()
        cobyla.set_options(maxiter=250)
        ry = TwoLocal(qubitOp.num_qubits, 'ry', 'cz', reps=5, entanglement='full')
        vqe = VQE(qubitOp, ry, cobyla)
        vqe.random_seed = seed
        quantum_instance = QuantumInstance(backend=backend, seed_simulator=seed, seed_transpiler=seed)
        result = vqe.run(quantum_instance)
        return self.decode_result(result)

    def qaoa_solution(self):
        qubitOp = portfolio_diversification.get_operator(self.rho, self.n, self.q)
        backend = Aer.get_backend('statevector_simulator')
        seed = 50
        cobyla = COBYLA()
        cobyla.set_options(maxiter=250)
        qaoa = QAOA(qubitOp, cobyla, 3, 'matrix')
        qaoa.random_seed = seed
        quantum_instance = QuantumInstance(backend=backend, seed_simulator=seed, seed_transpiler=seed)
        result = qaoa.run(quantum_instance)
        return self.decode_result(result)

    def get_portfoliodiversification_solution(self, n, result):
        v = result.eigenstate
        if isinstance(v, StateFn):
            v = v.to_matrix()

        N = n ** 2 + n

        index_value = [x for x in range(len(v)) if v[x] == max(v)][0]
        string_value = "{0:b}".format(index_value)

        while len(string_value) < N:
            string_value = '0' + string_value

        x_state = list()
        for elements in string_value:
            if elements == '0':
                x_state.append(0)
            else:
                x_state.append(1)

        x_state = np.flip(x_state, axis=0)

        return x_state

    def decode_result(self, result, offset = 0):
        quantum_solution = self.get_portfoliodiversification_solution(self.n, result)
        ground_level = portfolio_diversification.get_portfoliodiversification_value(self.rho, self.n, self.q, quantum_solution)
        return quantum_solution, ground_level
